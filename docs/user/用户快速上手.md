# 用户快速上手

> 🎯 5 分钟快速上手 | 完整操作说明请参考 [完整操作手册](../完整操作手册.md)

## 一、快速入门 (5 分钟上手)

### Step 1: 登录

访问系统首页 → 输入用户名和密码 → 登录

### Step 2: 创建项目

项目管理 → 新建项目 → 填写：

| 字段 | 示例 |
|------|------|
| 项目名称 | 订单系统 |
| Git 地址 | `https://github.com/xxx/order-service.git` |
| 分支 | `main` |
| 语言 | Java |

### Step 3: 解析代码

项目列表 → 点击「开始解析」→ 等待进度 100%

```
克隆仓库 → 解析代码结构 → 写入 Neo4j → 解析 Git 历史 → 完成
  20%        50%            70%         90%           100%
```

### Step 4: 开始提问

智能问答 → 选择项目 → 输入问题

## 二、智能问答速查

### 支持的问法类型

| 类型 | 问法示例 | 返回信息 |
|------|---------|---------|
| 🔗 调用链查询 | "谁调用了 `createOrder`？" | 调用者列表 + 文件位置 |
| 💥 影响分析 | "修改 `validateOrder` 会影响什么？" | 受影响方法 + 影响深度 |
| 🔍 代码溯源 | "`createOrder` 是谁写的？" | 作者 + 提交信息 + 时间 |
| 📋 类方法查询 | "`OrderService` 有哪些方法？" | 方法列表 + 签名 + 位置 |
| 📜 修改历史 | "`createOrder` 的修改历史？" | 提交记录时间线 |
| 🏗️ 服务依赖 | "订单服务依赖哪些服务？" | 依赖服务列表 |

### 提问技巧

1. **用引号包裹名称** → `"谁调用了 createOrder？"`
2. **指定完整类名** → `OrderService.createOrder` 避免重名歧义
3. **组合条件** → `"张三上周修改的 createOrder 相关代码"`
4. **点击来源链接** → 跳转到具体代码行

## 四、常见问题速查

| 问题 | 解决方案 |
|------|---------|
| 解析失败 | 检查 Git 地址、分支、访问权限 |
| 问答不准 | 确认项目已解析完成 → 触发增量同步 → 使用更完整的方法名 |
| 历史对话 | 左侧「历史对话」列表自动保存 |
| Webhook 配置 | 项目详情 → Webhook → 填写 URL 和 Secret |
