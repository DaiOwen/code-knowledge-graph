## 详细结论

### TV-01: MCP Java SDK

- **SDK**: `io.modelcontextprotocol:mcp` (官方 SDK)
- **连接方式**: Stdio Transport (子进程通信)
- **关键发现**: SDK 成熟可用，支持 stdio 和 SSE 两种传输方式
- **备选方案**: 自建 Stdio 封装 (如 SDK 遇到问题)

### TV-02: codegraph 输出格式

- **格式**: Markdown 文本
- **解析方式**: 正则表达式提取
- **关键发现**:
  - 输出包含符号名、类型、文件路径、行号
  - `codegraph_explore` 返回调用链和源代码
  - `codegraph_callers/callees` 返回调用关系
- **测试结果**: 在测试项目上成功解析

### TV-03: Neo4j 批量写入

- **策略**: UNWIND 批量插入 + 分批事务
- **批次大小**: 1000 节点/批
- **幂等性**: 使用 MERGE 代替 CREATE
- **性能预估**: 10 万节点 + 50 万关系 < 5 分钟

### TV-04: GraphRAG 端到端

- **流程**: 问题理解 → 查询规划 → Cypher 执行 → 答案生成
- **LLM 框架**: LangChain4j
- **模板库**: 6 种基础查询模板
- **流式输出**: SSE (Server-Sent Events)
